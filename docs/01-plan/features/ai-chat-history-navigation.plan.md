## 1. 요구사항 (Requirements)

### 1.1 히스토리 기록
- [ ] 메시지 전송(Submit) 성공 시, 입력값(trimmed)을 히스토리 배열에 저장
- [ ] 동일한 메시지를 연속으로 보낼 경우 히스토리 중복 저장 방지
- [ ] 히스토리는 세션 동안 유지 (브라우저 메모리 상의 배열)

### 1.2 위쪽 화살표 (ArrowUp) 탐색
- [ ] 입력창에서 `ArrowUp` 키 입력 시, 현재보다 한 단계 이전의 메시지를 입력창에 로드
- [ ] 히스토리 탐색 시작 시, 현재 입력 중이던 텍스트를 임시 저장(Stash)하여 나중에 복구 가능하게 함
- [ ] 히스토리의 끝(가장 오래된 메시지)에 도달하면 더 이상 변하지 않음

### 1.3 아래쪽 화살표 (ArrowDown) 탐색
- [ ] 히스토리 탐색 중 `ArrowDown` 키 입력 시, 다음(최신 쪽) 메시지를 로드
- [ ] 가장 최신 메시지에서 `ArrowDown`을 누르면 이전에 임시 저장했던 텍스트(Stash)를 복구
- [ ] 히스토리 탐색 중이 아닐 때는 기본 동작(커서 이동) 유지

### 1.4 예외 상황 처리
- [ ] 입력창이 비어있을 때도 히스토리 탐색 가능
- [ ] 엔터 키로 전송 시 히스토리 인덱스 초기화
- [ ] (선택 사항) 다중 행 입력 모드(Shift+Enter)일 때, 커서가 첫 줄에 있을 때만 Up, 마지막 줄에 있을 때만 Down 동작하도록 정교화 (1단계에서는 단순 키 이벤트 우선 적용)

## 2. 설계 (Design)

### 2.1 데이터 구조 (State)
```typescript
const [history, setHistory] = useState<string[]>([]); // 보낸 메시지 목록
const [historyIndex, setHistoryIndex] = useState(-1); // 현재 탐색 중인 위치 (-1: 탐색 미수행)
const [stash, setStash] = useState(''); // 탐색 전 입력값 임시 보관
```

### 2.2 주요 로직 (Handlers)
- **onKeyDown**:
  - `ArrowUp`: 
    - `historyIndex === -1`이면 `stash`에 현재 `value` 저장
    - `historyIndex < history.length - 1`이면 인덱스 증가 및 `value` 업데이트
  - `ArrowDown`:
    - `historyIndex > -1`이면 인덱스 감소
    - `historyIndex === -1`이 되면 `value`를 `stash`로 복구
    - 그 외(`historyIndex >= 0`)면 `value` 업데이트
- **handleSubmit**:
    - 전송 성공 시 `history` 업데이트 (`history[0]`과 다를 때만 추가)
    - `historyIndex`를 `-1`로 리셋
    - `stash`를 `''`로 리셋

## 4. 성공 기준 (Success Criteria)

- [ ] SC-1: 메시지 전송 후 위쪽 화살표를 누르면 해당 메시지가 다시 나타남
- [ ] SC-2: 여러 번 전송 후 위쪽 화살표를 연속으로 누르면 역순으로 메시지 탐색 가능
- [ ] SC-3: 탐색 중 아래쪽 화살표를 누르면 최신 쪽으로 이동하며, 마지막엔 원래 쓰던 글이 복구됨
- [ ] SC-4: 엔터 전송 후 히스토리 탐색이 초기 상태(-1)에서 시작됨

## 5. 리스크 및 대응

- **텍스트 유실**: 히스토리 탐색 중 실수로 다른 글을 덮어쓰지 않도록 `stash` 기능을 엄격히 구현
- **UX 충돌**: 텍스트가 여러 줄일 때 위로 이동하려다 히스토리가 바뀌는 문제 -> 초기 버전은 단순 키 이벤트로 구현하고, 필요시 커서 위치 체크 로직 추가
