# Plan: ai-panel-auto-collapse

> **Feature**: AI 패널 자동 접힘 (캘린더 클릭 연동)  
> **Created**: 2026-05-13  
> **Phase**: Plan  

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | AI 어시스턴트 패널이 열려 있을 때 화면의 절반 이상을 차지하여 캘린더의 일정을 자세히 확인하거나 조작하기 불편함 |
| **Solution** | 캘린더 영역(날짜 슬롯, 이벤트 등) 클릭 시 AI 패널을 자동으로 접는(Collapse) 로직 구현 |
| **Functional UX Effect** | AI와 대화하다가 직접 캘린더를 조작하고 싶을 때 별도의 '닫기' 버튼 클릭 없이 즉시 전체 화면 확보 가능 |
| **Core Value** | 직관적인 화면 전환을 통한 멀티태스킹 효율성 향상 |

---

## Context Anchor

| 항목 | 내용 |
|------|------|
| **WHY** | AI 패널의 높이가 높아 캘린더 가독성을 저해함. 사용자의 의도가 '캘린더 조작'으로 옮겨갔을 때 자동으로 공간을 확보해줄 필요가 있음 |
| **WHO** | 캘린더와 AI 어시스턴트를 병행 사용하는 사용자 |
| **RISK** | 의도치 않은 패널 닫힘으로 인한 대화 맥락 단절(Stash 기능으로 보완됨), 드래그 앤 드롭 등 복잡한 조작과의 충돌 |
| **SUCCESS** | 월간/주간/일간 뷰에서 슬롯 클릭 시 AI 패널이 `collapsed: true` 상태로 즉시 변경됨 |
| **SCOPE** | `DashboardPage`의 캘린더 클릭 핸들러 및 `useAiStore` 연동 |

---

## 1. 요구사항 (Requirements)

### 1.1 자동 접힘 트리거
- [ ] 월간 뷰(MonthView)의 날짜 슬롯 클릭 시 패널 접힘
- [ ] 주간/일간 뷰(WeekView)의 시간 슬롯 클릭 시 패널 접힘
- [ ] 스케줄 뷰(ScheduleView)의 항목 클릭 시 패널 접힘
- [ ] (선택 사항) 캘린더 내 빈 공간 클릭 시에도 패널 접힘

### 1.2 상태 유지 및 복구
- [ ] 패널이 접혀도 현재 입력 중이던 텍스트나 대화 내역은 유지되어야 함 (이미 구현됨)
- [ ] 수동으로 다시 열었을 때 이전 상태가 온전해야 함

### 1.3 예외 상황
- [ ] 패널 내부를 클릭하거나 패널 상단의 헤더 버튼을 조작할 때는 접히지 않아야 함
- [ ] 모달(EventModal)이 열려 있는 동안의 배경 클릭 처리는 기존 모달 로직을 따름

---

## 2. 기술적 설계 (Technical Design)

### 2.1 상태 제어
- `useAiStore`의 `setCollapsed(true)` 함수를 활용

### 2.2 구현 방식
- **Centralized Event Handling**: `DashboardPage`에서 캘린더 뷰들에게 전달하는 `onSlotClick`, `onEventClick` 등의 핸들러 내부에서 `setCollapsed(true)` 호출
- **Hook-based**: 필요한 곳에서 `useAiStore`를 구독하여 직접 처리

## 4. 성공 기준 (Success Criteria)

- [ ] SC-1: AI 패널이 열린 상태에서 캘린더 날짜 클릭 시 패널이 즉시 최소화됨
- [ ] SC-2: AI 패널이 이미 접혀 있는 상태에서는 클릭 시 아무런 부정적 영향이 없음
- [ ] SC-3: 캘린더 슬롯 클릭 후에도 기존의 '일정 생성 모달'은 정상적으로 열려야 함
- [ ] SC-4: AI 입력창에 쓰던 글이 패널이 접힌 후 다시 열었을 때 그대로 유지됨

---

## 5. 리스크 및 대응

- **UX 간섭**: 사용자가 단순히 캘린더를 훑어보는 행위(단순 클릭)만으로 패널이 닫히는 것이 불편할 수 있음. 하지만 현재 AI 패널의 높이가 매우 높으므로(55vh), 캘린더 클릭은 대개 캘린더 조작 의지로 판단하는 것이 합리적임.
- **모바일 대응**: 모바일에서는 특히 화면이 좁으므로 이 기능이 필수적임.
