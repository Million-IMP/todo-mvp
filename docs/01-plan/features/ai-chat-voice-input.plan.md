# Plan: ai-chat-voice-input

> **Feature**: AI 채팅 음성 입력 (Web Speech API)  
> **Created**: 2026-05-12  
> **Phase**: Plan  

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 이동 중이거나 타이핑이 어려운 상황에서 AI에게 일정을 입력하기 불편함 |
| **Solution** | 브라우저 표준 Web Speech API를 활용하여 음성 인식(Speech-to-Text) 기능을 AI 입력창에 통합 |
| **Functional UX Effect** | 버튼 클릭 한 번으로 말하듯이 자연스럽게 일정을 추가하거나 검색할 수 있음 |
| **Core Value** | "Hands-free" 입력 지원을 통한 접근성 및 편의성 극대화 |

---

## Context Anchor

| 항목 | 내용 |
|------|------|
| **WHY** | AI 어시스턴트의 핵심 가치인 '자연스러운 대화'를 실현하기 위해 음성 인터페이스 필요 |
| **WHO** | 빠른 입력이 필요한 모든 사용자, 접근성 지원이 필요한 사용자 |
| **RISK** | 브라우저별 호환성 차이 (Chrome/Edge 권장), 소음 환경에서의 인식률 저하, 마이크 권한 거부 |
| **SUCCESS** | 음성 버튼 클릭 시 녹음 시작 -> 말하기 -> 텍스트 변환 후 입력창에 자동 삽입 정상 동작 |
| **SCOPE** | `AiInput.tsx` UI 수정 및 음성 인식 로직 구현 |

## 1. 요구사항 (Requirements)

### 1.1 음성 인식 제어
- [ ] 입력창 옆에 마이크 아이콘 버튼 추가
- [ ] 버튼 클릭 시 음성 인식 시작/중단 토글
- [ ] 인식 중임을 알리는 시각적 피드백 (버튼 색상 변경 또는 애니메이션)
- [ ] 브라우저에서 마이크 권한 요청 처리

### 1.2 텍스트 변환 및 입력
- [ ] 실시간으로 변환된 텍스트를 입력창에 표시 (중간 결과 포함)
- [ ] 인식이 완료되면 최종 텍스트를 입력창의 현재 값에 추가 또는 교체
- [ ] (선택 사항) 특정 키워드(예: "전송해줘") 인식 시 자동 전송 기능

### 1.3 브라우저 호환성 및 에러 처리
- [ ] Web Speech API를 지원하지 않는 브라우저(Safari 일부 버전 등)에서는 버튼 숨김 처리
- [ ] 인식 에러 발생 시 사용자에게 알림 (토스트 또는 아이콘 상태 표시)
- [ ] 언어 설정: 기본값 'ko-KR' (한국어) 적용

---

## 2. 기술적 설계 (Technical Design)

### 2.1 사용 API
- **Web Speech API (SpeechRecognition)**: 브라우저 내장 기능을 사용하여 별도의 서버 비용 없이 구현

### 2.2 구현 방식
- **Custom Hook (`useSpeechToText`)**: 음성 인식 로직을 캡슐화하여 재사용 가능하게 구현
- **AiInput 통합**: 훅에서 반환하는 `isListening`, `transcript`, `start`, `stop` 등을 UI에 바인딩

## 4. 성공 기준 (Success Criteria)

- [ ] SC-1: 마이크 버튼 클릭 시 마이크 권한 요청 팝업이 뜸
- [ ] SC-2: 음성 인식 중 버튼 색상이 빨간색으로 변경되거나 깜빡임
- [ ] SC-3: 한국어로 말했을 때 텍스트로 정확히 변환되어 입력창에 들어감
- [ ] SC-4: 인식 중 중단 버튼을 누르면 즉시 멈춤
- [ ] SC-5: 지원되지 않는 브라우저에서는 기능이 노출되지 않음

---

## 5. 리스크 및 대응

- **브라우저 호환성**: `window.SpeechRecognition` 또는 `window.webkitSpeechRecognition` 모두 지원하도록 대응
- **인식 정확도**: 한국어(`ko-KR`) 설정을 명시적으로 지정
- **UI 충돌**: 기존 히스토리 탐색 기능과 함께 사용해도 문제없도록 상태 관리 격리
