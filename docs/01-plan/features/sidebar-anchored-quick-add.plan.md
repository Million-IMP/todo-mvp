# Plan: sidebar-anchored-quick-add

> **Feature**: 사이드바 앵커형 일정 생성 UI
> **Created**: 2026-05-14
> **Phase**: Plan

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 메인 캘린더 클릭 시 중앙 모달이 떠서 작업 맥락이 끊기고 시각적으로 무거움 |
| **Solution** | 일정 생성 창을 사이드바의 미니 캘린더 하단 영역에 팝오버 형태로 배치 |
| **Functional UX Effect** | 구글 캘린더와 유사한 '사이드바 퀵 메뉴' 느낌을 주어 효율적인 입력 경험 제공 |
| **Core Value** | 화면 공간의 효율적 활용 및 시각적 일관성 확보 |

---

## 1. 요구사항 (Requirements)

### 1.1 위치 고정
- [ ] 메인 캘린더 슬롯(날짜/시간) 클릭 시, 입력창이 사이드바 미니 캘린더 바로 아래에 렌더링되어야 함.
- [ ] 사이드바가 닫혀 있는 경우, 자동으로 사이드바를 열거나 적절한 기본 위치(왼쪽 상단 등)에 표시.

### 1.2 UI 전환
- [ ] 기존 `EventModal`을 재사용하거나, 사이드바 위치에 최적화된 `QuickAddPopover` 구현.
- [ ] 화살표(Tail)가 포함된 팝오버 스타일 적용 가능 여부 검토.

### 1.3 기능 유지
- [ ] 제목, 날짜, 시간, 카테고리 등 기존 `EventModal`의 모든 편집 기능 유지.

---

## 2. 구현 계획

### 2.1 앵커 포인트 설정
- `Sidebar.tsx`의 미니 캘린더 컨테이너에 `useRef`를 사용하여 위치 정보(Rect)를 추출.
- `DashboardPage.tsx`에서 이 위치 정보를 관리하고 생성 창에 전달.

### 2.2 컴포넌트 수정
- `EventModal.tsx`: `anchorRect` prop을 추가하여, 값이 있으면 해당 위치에 팝오버로 렌더링하고 없으면 기존처럼 중앙 모달로 작동하도록 개선.

---

## 3. 성공 기준 (Success Criteria)

- [ ] SC-1: 메인 캘린더 클릭 시 입력창이 사이드바 미니 캘린더 하단에 정확히 정렬되어 나타남.
- [ ] SC-2: 입력창이 열린 상태에서도 메인 캘린더의 시야를 가리지 않음.
- [ ] SC-3: '저장' 또는 '취소' 시 정상적으로 닫히고 데이터가 반영됨.

---

## 4. 구현 순서

1. **Phase 1 - 앵커 감지**: `Sidebar`에서 미니 캘린더 하단 위치를 계산하여 상위로 보고하는 로직 추가.
2. **Phase 2 - UI 포지셔닝**: `EventModal`을 팝오버 형태로 렌더링할 수 있도록 CSS 및 로직 수정.
3. **Phase 3 - 통합 및 조정**: 사이드바 개폐 여부에 따른 예외 처리 및 최종 레이아웃 조정.
